# Support

Thanks for your interest in the Clearance Manifest standard. Here is where to go
for each kind of question.

## Where to go

| You want to... | Go here |
|---|---|
| Report a bug in the spec | [Open an issue](https://github.com/openclearance/openclearance/issues/new). A "bug" here means the prose, a JSON Schema, the JSON-LD context, the rule registry, or an example fixture disagree with one another, or a published artifact is wrong. Include the artifact, what you observed, and what you expected. |
| Report a security or integrity vulnerability | Do not open a public issue. Follow [SECURITY.md](SECURITY.md). Spec ambiguities that enable forgery or audit-defeat, and schemas that accept a document the spec forbids, are highest priority. |
| Read the spec | The current line is published at [openclearance.org/v0.1/](https://openclearance.org/v0.1/): the normative `spec`, the `rules` registry, the `vocab` term map, and the machine artifacts (`context.jsonld`, the schemas, the conformance examples) under `/v0.1/`. |
| Propose a new field, rule, or vocabulary term | This is a spec change. Use the [spec-change-proposal issue form](https://github.com/openclearance/openclearance/issues/new?template=spec-change-proposal.yml) and read [GOVERNANCE.md](GOVERNANCE.md) first: it defines what counts as a patch, minor, or major change and what each requires. |
| Weigh in on long-term governance | The governance model is intentionally an open question. Comment on the pinned **"Long-term governance model"** issue. See [GOVERNANCE.md](GOVERNANCE.md). |
| Ask an open-ended question | Open an issue (or a GitHub Discussion if Discussions is enabled on the repo), or email **contact@openclearance.org**. |

## Response expectations

The standard has a single maintainer (see [MAINTAINERS.md](MAINTAINERS.md) and
[GOVERNANCE.md](GOVERNANCE.md)). There is no SLA for bug reports or feature
requests, but pull requests get reviewed promptly. Security reports get an
acknowledgement within 7 days, per [SECURITY.md](SECURITY.md).

If you can write the fix yourself, a pull request is the fastest path. See
[CONTRIBUTING.md](CONTRIBUTING.md).
